Problem:
AI agents are starting to do real work for each other (generating data, running analyses, completing tasks). But there's zero infrastructure for safe payments between them. If Agent A pays Agent B and Agent B delivers crap, the money is just gone. No refunds, no disputes, no accountability. This means agents can only transact with agents they already trust, which makes an open agent economy impossible. It's like trying to run eBay with no buyer protection in 2003.
Solution:
An autonomous escrow and dispute resolution protocol for AI agent commerce. Instead of paying each other directly, agents lock funds in a TRON escrow contract. The seller agent uploads deliverables and work logs to Filecoin as tamper-proof evidence. If the buyer is satisfied, funds release. If not, a third-party arbitrator agent reviews the Filecoin-stored evidence and makes a binding ruling on-chain. Every outcome (good delivery, bad delivery, dispute won/lost) feeds into a Filecoin-anchored reputation score that follows the agent everywhere. Over time, trustworthy agents get better terms (lower escrow requirements) and sketchy agents get priced out.
Why Filecoin:
You need a place to store the evidence that nobody can tamper with. If the deliverables and work logs lived on a centralized server, either side could edit them after the fact to win a dispute. Filecoin gives you CID-anchored, permanent, verifiable storage. The arbitrator agent doesn't have to trust either party because it's pulling evidence from an immutable source. Reputation histories also live on Filecoin, so agents carry a portable, unfakeable track record across any platform. Remove Filecoin and your dispute system has no trustworthy evidence, which makes it useless.
Why TRON:
Two things no other chain gives you this cleanly. First, TRON's native Account Permission Management lets you constrain agent wallets at the protocol level: the arbitrator can only call dispute resolution functions, agents can't overspend, large transactions require multi-sig approval. This isn't a smart contract workaround, it's built into TRON's architecture. Second, TRON is the dominant USDT settlement layer with near-zero fees. Agent micro-escrows (locking up $0.50 for a small task) need to be cheap enough that the fee doesn't eat the transaction. On Ethereum L1 that's laughable. On TRON it actually works.
Why we win the hackathon:
Three reasons. First, "agentic chargeback / dispute management" is explicitly listed as a track direction in the TRON bounty and almost nobody will build it because it's harder than a basic payment flow. Blue ocean. Second, both chains are structurally essential, not cosmetically attached. Judges can't look at this and say "you could swap TRON for Solana" or "Filecoin is just a nice-to-have." The escrow needs TRON's permission system, the dispute needs Filecoin's immutable evidence. Third, the demo is visceral: judges watch two agents make a deal, one delivers bad work, a dispute gets filed, an arbitrator reviews evidence, funds get refunded, and the bad agent's reputation drops in real time. That's a story, not just a tech demo.
